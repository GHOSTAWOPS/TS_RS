# TS_RS UI 决策记录

本文档记录每轮 grill-me 讨论后已经确认的 UI 决策。

原则：每轮只确认一个关键分支；确认后立即追加到本文档，避免后续开发时遗忘或跑偏。

## 当前 UI 总路线

TS_RS 是独立 Qt6 + OCCT 桌面程序，不追求 VisualTS、RebarSmart 3DE 或 3DE 原生界面的像素级复刻。

当前路线：

```text
文件 / 视图：参考图石和独立 Qt/OCCT CAD 程序需要
钢筋生成：参考 RebarSmart 的筋型、参数、生成流程
工程图 / Detail：参考老图石 Detail 包和 CAD 插件兼容流程
```

暂不使用 RebarSmart 的这些模块作为主线：

```text
钢筋校审
钢筋导出
钢筋切图
```

这些能力如果后续需要，只能作为独立阶段重新评估，不能混进当前 MVP 主线。

## DEC-UI-001：主界面不是老图石像素级复刻

问题：

新系统 UI 是否要 1:1 复刻老 VisualTS / 图石 MFC 界面。

用户结论：

不做老界面像素级复刻，但主菜单感觉可以参考图石，尤其是上方菜单栏和模块分类。

最终决策：

采用现代 Qt Widgets CAD 布局：

```text
QMainWindow
顶部菜单 / 工具栏
中央 OCCT Viewer
左侧工程树 / 模型树 / 钢筋树
右侧当前钢筋生成器参数 Dock
底部消息 / 选择 / 下料表 / 诊断 Dock
```

证据来源：

- 用户确认：图石上部菜单栏更符合使用习惯。
- 用户确认：老系统界面很丑，不要求像素级照搬。

影响范围：

- UI 层可以重新设计。
- 业务逻辑和参数体系不能因为 UI 现代化而偏离 RebarSmart / 图石证据。

## DEC-UI-004：钢筋组右键作为已生成钢筋的主编辑入口

问题：

已生成钢筋和钢筋组的修改入口应放在顶层“编辑”页签，还是放在左侧 Dock 右键菜单。

用户结论：

采用方案 A：

```text
钢筋组右键作为主要编辑入口；
单根钢筋右键先只做定位、高亮、信息、选中所属组；
单根改参/删除先记为 P1 待确认。
```

最终决策：

左侧 Dock 的钢筋树提供钢筋组右键菜单。

单根钢筋右键保持保守，不直接破坏参数化钢筋组。

证据来源：

- RebarSmart 说明书 4.3 钢筋编辑：已创建钢筋组可全参数编辑、批量更新、生成/删除实体、修改等级/直径/备注/编号等。
- RebarSmart 资源文件：
  - `EditRebarFullParametersDlg.CATNls`
  - `ModifyRebarDiameterDlg.CATNls`
  - `DuplicateRebarTranslateDlg.CATNls`
  - `RebarShowHideByAllDlg.CATNls`
  - `ShowRebarInfoDlg.CATNls`
  - `SearchRebarDlg.CATNls`
- IDA MCP：
  - `RebarEditing.dll` 可打开，确认存在 `fctCreateModifyRebarDiameterCmd`、`fctCreateModifyRebarGradeCmd`、`fctCreateModifyRebarSpaceCmd`、`fctCreateEditRebarSolidCreateCmd`、`fctCreateEditRebarSolidDeleteCmd`。
  - `RebarShowHide.dll` 可打开，确认存在 `fctCreateRebarShowHideByAllCmd`、`fctCreateRebarShowHideByAssignCmd`、`fctCreateRebarShowHideByCountCmd`、`fctCreateRebarShowHideByHeadTailCmd`、`fctCreateRebarShowHideByScopeCmd`、`fctCreateRebarShowHideBySpaceCmd`。
  - `RebarReview.dll` 可打开，确认存在 `fctCreateShowRebarInfoCmd`、`fctCreateSearchRebarCmd`，并出现 `IsRebarObj`、`IsRebarGSRoot`、`GetRebarGSRoot`、`GetRebarCount` 等符号。
  - `RebarNumbering.dll` 可打开，确认存在 `fctCreateRebarNumberingModeAssignCmd`、`fctCreateRebarNumberingModeAutoCmd`、`fctCreateRebarNumberingModeBatchCmd`、`fctCreateRebarNumberingModeHierachyCmd`，并出现 `GetValRebarGSNo`、`SetValRebarGSNo`、`GetValRebarObjNo`、`SetValRebarObjNo` 等符号。

边界说明：

RebarSmart 证据证明它有钢筋组编辑、显隐、查找、信息查看等能力。

但当前证据没有证明 RebarSmart 原生 UI 就是右键菜单。把这些能力映射到左侧 Dock 右键菜单，是 TS_RS 的新 UI 交互设计。

待确认：

- 单根钢筋是否允许独立删除、独立改参，需要后续结合 RebarSmart 运行或 IDA 深挖确认。

## DEC-UI-005：单根钢筋右侧 Dock 只做信息和快捷操作

问题：

单根钢筋被选中后，右侧是否也需要功能菜单。

用户问题：

```text
所以单根钢筋有没有右侧功能菜单
```

最终决策：

有，但 P0 只做“单根钢筋信息 / 快捷操作”面板，不做完整编辑面板。

右侧 Dock 可显示：

```text
定位
高亮
显示钢筋信息
选中所属钢筋组
显示 / 隐藏
查看属性
```

右侧 Dock 不显示：

```text
修改直径
修改等级
修改间距
删除单根
编辑全部参数
```

理由：

RebarSmart 已确认的强编辑能力主要绑定到“钢筋组 / 钢筋几何集 / 编号范围”。单根钢筋更适合作为查看、高亮、定位和反查所属组的选择对象。

如果单根钢筋直接支持改参或删除，TS_RS 必须引入“组内局部 override”模型，这会显著增加领域模型复杂度，不适合作为 P0。

影响范围：

- 选择单根钢筋时，右侧 Dock 不空白。
- 右侧 Dock 应提示其所属钢筋组，并提供“跳转到钢筋组编辑”的入口。
- 完整修改仍由钢筋组参数面板承接。

## DEC-UI-007：开始页签移除 Detail 导出，撤销重做降为架构预留

问题：

开始页签是否放 `导出 Detail 包`、`撤销`、`重做`。

用户结论：

```text
导出 Detail 包放到工程图里。
撤回重做操作这个要考虑下技术可行性能不能做。
```

最终决策：

`导出 Detail 包` 不放在开始页签，只放在工程图页签。

`撤销 / 重做` 技术上可行，但不作为 P0 必做按钮。

原因：

Qt6 Widgets 有 `QUndoCommand`、`QUndoStack`、`QUndoGroup`、`QUndoView` 等撤销框架，可以把用户操作封装成可 undo / redo 的命令。

但 TS_RS 的修改对象包括工程文件、STEP 绑定、钢筋组参数、OCCT 显示对象、工程图输出等。只有当这些修改都经过统一 command service 时，撤销/重做才可靠。

阶段结论：

```text
P0：保留 CommandService / Transaction / DirtyState 设计口径，不承诺全功能撤销。
P1：先支持轻量撤销，例如钢筋组新增、删除、参数修改、显隐。
P2：再评估工程图生成、STEP 导入、Detail 导出是否纳入撤销。
```

## DEC-UI-008：参考面 / 参考线可做，作为参考几何能力进入设计

问题：

参考面、参考线能不能做。

结论：

能做，而且后续钢筋创建会需要。

证据：

RebarSmart 说明书中出现：

```text
轴线延长 / 截断到参考曲面
选择首端延长参考方向及长度
选择末端延长参考方向及长度
用户绘制好的钢筋轴线
用户选择绘制好的钢筋轴线
```

技术判断：

OCCT 可以表达参考几何：

```text
参考点：gp_Pnt
参考线 / 轴线：gp_Ax1、Geom_Line、TopoDS_Edge
参考面：gp_Ax2、Geom_Plane、TopoDS_Face
显示选择：AIS_Shape 或自定义 AIS_InteractiveObject
```

阶段结论：

```text
P0：支持从 STEP 面/边提取参考面、参考线；支持在当前命令内临时选择参考方向。
P1：支持用户新建持久参考面、参考线，并显示在左侧工程树。
P2：支持复杂基准几何管理，例如偏移参考面、两点建线、三点建面、参考几何编辑。
```

UI 入口建议：

```text
钢筋页签：放“定义平面 / 参考线定义”工具组。
钢筋命令右侧 Dock：在需要时提供“选择参考面 / 选择参考线 / 选择参考方向”步骤。
显示页签：只保留参考几何的显示 / 隐藏，不作为创建入口。
```

## DEC-UI-010：定义平面 / 参考线定义第一版采用小窗口

问题：

用户结论：

```text
小窗口
```

最终决策：

第一版采用小窗口。

理由：

```text

2. 定义平面 / 参考线定义是辅助几何命令，不是长流程钢筋生成器。
```

窗口要求：

```text
小窗口必须支持预览 / 确定 / 取消。
拾取对象时允许窗口临时保持打开。
生成结果写入参考几何模型，不能只是临时 AIS 显示。
```

## DEC-UI-011B：显示页签参考图石分组

问题：

显示页签应如何组织按钮。

用户提供图石显示截图：

```text
C:\Users\ghost\Desktop\PixPin_2026-06-11_16-59-27.png
```

截图可见分组：

```text
取景
方位
渲染
钢筋显示
切平面
窗口
```

最终决策：

TS_RS 显示页签参考图石分组，但用 OCCT Viewer 实现。

P0 分组：

```text
取景：
  全显
  实时缩放
  实时平移
  窗口缩放

方位：
  轴测
  俯视 / 正视 / 侧视
  垂直于
  旋转

渲染：
  线框
  渲染
  上色

钢筋显示：
  激活
  钝化

切平面：
  X 向剖视
  Y 向剖视
  Z 向剖视
  设置旋转
```

说明：

`垂直于` 表示以当前选中面或参考平面为视图法向。

`旋转` 表示 OCCT Viewer 进入动态视图旋转状态，必须支持鼠标拖拽旋转，也可以提供工具栏按钮进入旋转模式。

`设置旋转` 属于切平面工具，表示把当前切平面旋转一定角度，不是视图旋转设置。

`激活 / 钝化` 解释为钢筋可见性状态：激活表示看得到，钝化表示看不到，语义接近显示 / 隐藏。

图石截图里的 `窗口重叠 / 窗口水平平铺 / 窗口垂直平铺` 不进入 TS_RS P0。第一版只做单个 OCCT Viewer，不做多窗口 / 多文档排列。

## DEC-UI-011E：切平面设置旋转语义

问题：

图石显示页签 `切平面 -> 设置旋转` 是什么含义。

用户结论：

```text
类似把切平面旋转一定角度
```

最终决策：

`设置旋转` 属于切平面工具，不属于视图方位工具。

语义：

```text
对当前 X / Y / Z 剖视生成的切平面进行角度旋转。
```

P0 口径：

```text
选择当前切平面
输入旋转角度
选择旋转轴或默认绕当前切平面中心轴
预览
确定
取消
```

P1：

```text
支持拖拽旋转手柄
支持多剖切面分别旋转
保存 / 恢复剖切面旋转状态
```

补充说明：

```text
这里的“设置旋转”只表示剖切面的角度调整。
OCCT Viewer 本身的视图旋转是 P0 必做能力，归入“方位 -> 旋转”和鼠标交互。
```

OCCT 能力确认：

```text
OCCT 支持 XYZ 剖切和剖切面旋转。

XYZ 剖切：
  使用 Graphic3d_ClipPlane + gp_Pln。
  X / Y / Z 剖视就是创建不同法向的 gp_Pln。
  ClipPlane 可以加到整个 V3d_View，也可以加到单个 AIS 对象。

剖切面旋转：
  保存当前 gp_Pln。
  用 gp_Trsf::SetRotation(gp_Ax1, angle) 或等价方式更新平面方向。
  再调用 Graphic3d_ClipPlane::SetEquation(newPlane)。
  最后刷新 V3d_View。
```

边界：

```text
这里确认的是 Viewer 显示剖切能力。
它不等于工程图 Detail 剖切线 / 隐藏线 / 标注生成算法已经完成。
```

## DEC-UI-024：钢筋创建和钢筋组右键统一切换右侧 Dock

问题：

钢筋标签页点击生成钢筋，以及钢筋组右键修改 / 复制 / 编辑，是弹窗还是右侧 Dock。

用户结论：

```text
统一右侧 Dock。
钢筋标签页和钢筋组右键都用 Dock。
```

最终决策：

右侧 Dock 是 TS_RS 的“当前钢筋操作面板”。

统一规则：

```text
钢筋标签页创建钢筋
  -> 切换右侧 Dock 到对应生成器面板

钢筋组右键编辑 / 复制 / 修改
  -> 切换右侧 Dock 到对应操作面板

单根钢筋查看 / 定位 / 信息
  -> 切换或保持右侧 Dock 的单根钢筋信息面板

定义平面 / 参考线定义 / 抽线
  -> 继续使用小窗口
```

交互原则：

```text
1. 右键菜单只负责选择命令，不直接承载复杂参数表。
2. 复杂参数、预览、应用、取消都放在右侧 Dock。
3. 钢筋创建和钢筋编辑都可能需要反复选择几何、调整参数、预览结果，Dock 比弹窗更稳定。
4. 定义平面 / 参考线定义属于短流程辅助几何工具，保留图石风格小窗口，避免挤占钢筋生成器 Dock。
```

钢筋组右键切换规则：

```text
选中钢筋组
  -> 右侧 Dock 显示钢筋组摘要面板

右键 -> 编辑全部参数
  -> 切到钢筋组全参数编辑面板

右键 -> 修改直径 / 等级 / 间距 / 备注
  -> 切到快速修改面板，目标钢筋组自动带入

右键 -> 平移复制 / 对称复制 / 旋转复制
  -> 切到复制操作面板，源钢筋组自动带入

右键 -> 生成实体 / 删除实体
  -> 切到实体操作确认面板

右键 -> 显示钢筋信息 / 查看属性
  -> 切到钢筋组信息面板

右键 -> 定位 / 高亮
  -> 直接执行视图动作，不强制切换 Dock

右键 -> 显示 / 隐藏
  -> P0 可直接执行；如后续需要范围选项，再切换到显隐面板
```

钢筋标签页创建规则：

```text
点击定距钢筋
  -> 右侧 Dock 切到 FixDistancePanel

点击定数钢筋
  -> 右侧 Dock 切到 FixNumberPanel

点击角筋 / 拉结筋 / 插筋 / 柱锥环盘面筋 / 箍筋 / 自定义钢筋
  -> 右侧 Dock 切到对应专用生成器面板
```

生成器应用后的停留规则：

```text
点击“应用”并成功生成一组钢筋后：
  -> 右侧 Dock 保留当前生成器
  -> 保留上一次参数值
  -> 清空已选择的几何对象 / 目标对象
  -> 清除临时预览
  -> 状态回到“等待选择对象”

目的：
  -> 方便连续创建同类型钢筋
  -> 避免用户每次重复输入直径、等级、间距、保护层等常用参数
```

边界：

```text
1. UI handler 不允许直接写钢筋算法。
2. Dock 面板只负责参数、选择状态、预览和命令提交。
3. 真正生成 / 修改 / 复制逻辑由 generator / service / domain 层承接。
4. 单根钢筋 P0 仍不提供完整改参 / 删除，只做信息、定位、高亮、跳转所属组。
```
