## 一句话结论

```text
TS_RS 是独立 Qt6 + OCCT 桌面程序。
主界面外壳优先对齐老图石 Ribbon 页签。
钢筋创建参数和生成流程以 RebarSmart 弹窗为准。
工程图 / Detail 输出参考老图石。
```

## UI 技术路线

P0 采用：

```text
Qt Widgets
QMainWindow
顶部菜单 / 工具栏
中央 OCCT Viewer
左侧 Dock：工程树 / 模型树 / 钢筋树 / 图纸树
右侧 Dock：当前钢筋操作面板 / 当前选择信息
底部 Dock：消息 / 选择结果 / 下料表 / 错误诊断
```

暂不采用：

```text
Qt Quick / QML 主界面
老 VisualTS 像素级复刻
RebarSmart 3DE 原生界面复刻
AutoCAD 插件界面作为主 UI
```

P0 顶层页签固定为：

```text
开始
显示
钢筋
查询

```

不做独立顶层页签：

```text
选择
编辑
钢筋校审
钢筋导出
钢筋切图
设置
```

说明：

```text
选择由 OCCT Viewer + 当前命令承接。
编辑由左侧钢筋树右键 + 右侧 Dock 承接。
查询页签 P0 保留外壳，可先置灰或只放占位入口。
设置不做 P0 顶层页签，后续放到开始 / 工程 / 工具菜单或右上角选项。
RebarSmart 的校审 / 导出 / 切图不作为当前主线。
```

## 显示页签

P0 参考图石显示页签分组，但由 OCCT Viewer 实现：

```text
取景：
  全显
  实时缩放
  实时平移
  窗口缩放

方位：
  轴测
  俯视 / 正视 / 侧视
  垂直于
  旋转

渲染：
  线框
  渲染
  上色

钢筋显示：
  激活
  钝化

切平面：
  X 向剖视
  Y 向剖视
  Z 向剖视
  设置旋转
```

语义：

```text
激活 = 显示钢筋
钝化 = 隐藏钢筋
旋转 = OCCT Viewer 视图旋转
设置旋转 = 当前剖切面旋转角度
```

P0 不做：

```text
窗口重叠
窗口水平平铺
窗口垂直平铺
隐线不可见
隐线可见
多剖切面组合
剖切面拖拽手柄
保存 / 恢复剖视状态
```

## OCCT Viewer

P0 必做：

```text
STEP / STP 显示
鼠标旋转
平移
缩放
窗口缩放
适应窗口
点 / 边 / 面 / 实体选择
选择高亮
悬停高亮
X / Y / Z 剖视
清除剖视
```

选择方式：

```text
不做独立“选择”页签。
在 Viewer 或命令状态中切换点 / 边 / 面 / 实体选择过滤。
当前命令需要什么对象，Dock 显示选择步骤和选择结果。
```

颜色语义：

```text
STEP / STP 结构模型：水泥灰
钢筋轴线 / 钢筋实体：橙黄
点 / 线 / 面选中：红色
当前高亮 / 定位 / 查找结果：洋红
参考几何：青色或黄色半透明
剖切面：半透明蓝灰
```

P0 不读取 STEP 原始颜色作为主显示颜色。

## 钢筋创建 Dock

所有钢筋创建命令点击后切换右侧 Dock：

```text
定距钢筋 -> FixDistancePanel
定数钢筋 -> FixNumberPanel
角筋 -> CornerPanel
拉结筋 -> TiePanel
插筋 -> DowelPanel
柱锥环盘面筋 -> FrustumPanel
箍筋 -> StirrupPanel
自定义钢筋 -> CustomizePanel
```

硬规则：

```text
每个钢筋创建 Dock 面板必须与 RebarSmart 对应 GenerateRebarXXXDlg 弹窗保持一致。
```

一致范围：

```text
参数分组
字段槽位
选择对象类型
默认值 / 单位 / 枚举语义
字段联动
校验逻辑
```

允许变化：

```text
弹窗 -> 右侧 Dock
确定 -> 应用
3DE 选择 -> OCCT Viewer 选择
增加顶部摘要 / 错误提示 / 折叠调试区
```

禁止：

```text
随便删字段
随便合并分组
自研简化参数体系
只做“够用”的简化版钢筋生成器
没有证据就改字段含义、默认值、单位或枚举
```

## Dock 顶部摘要

每个钢筋创建 / 编辑 Dock 顶部固定显示四行摘要：

```text
当前命令
当前对象
选择状态
状态提示
```

示例：

```text
当前命令：定数钢筋
当前对象：新建钢筋组
选择状态：布筋面 2，引导线 1，参考线 0
状态提示：可预览
```

调试信息默认折叠：

```text
对象 ID
commandId
selectedFaceIds
selectedEdgeIds
generatorClass
evidenceStatus
gapStatus
```

边界：

```text
顶部摘要不能替代 RebarSmart 原始参数分组。
当前对象不能只显示内部 ID，必须显示用户能理解的钢筋组 / 单根 / 新建状态。
```

## Dock 操作规则

钢筋创建 Dock：

```text
预览
应用
取消
```

应用成功后：

```text
保留当前生成器面板
保留上一次参数值
清空已选择几何对象 / 目标对象
清除临时预览
状态回到“等待选择对象”
```

取消：

```text
退出当前生成器
清除临时预览
右侧 Dock 回到当前选择对象摘要或空闲状态
```

## 钢筋树右键

左侧钢筋树是已生成钢筋的主编辑入口。

钢筋组右键 P0：

```text
定位
高亮
编辑全部参数
修改编号
修改等级
修改直径
修改间距
修改备注
显示 / 隐藏
平移复制
对称复制
旋转复制
生成钢筋实体
删除钢筋实体
显示钢筋信息
查找相关钢筋
查看属性
```

右键菜单只负责选择命令，复杂参数统一切换到右侧 Dock。

切换规则：

```text
编辑全部参数 -> 钢筋组全参数编辑面板
修改编号 / 等级 / 直径 / 间距 / 备注 -> 快速修改面板
平移复制 / 对称复制 / 旋转复制 -> 复制操作面板
生成实体 / 删除实体 -> 实体操作确认面板
显示钢筋信息 / 查看属性 -> 钢筋组信息面板
定位 / 高亮 -> 直接执行 Viewer 动作
显示 / 隐藏 -> P0 可直接执行
```

单根钢筋右键 P0：

```text
定位
高亮
显示钢筋信息
选中所属钢筋组
显示 / 隐藏
查看属性
```

单根钢筋 P0 不提供：

```text
修改直径
修改等级
修改间距
删除单根
编辑全部参数
```

原因：

```text
单根直接改参会破坏钢筋组参数化关系。
后续如要做，必须先设计组内局部 override 模型。
```

## 参考几何

参考几何入口放钢筋页签：

```text
定义平面
参考线定义
抽线
```

交互形式：

```text
小窗口
```

原因：

```text
参考几何是短流程辅助几何工具。
钢筋生成器和编辑器使用右侧 Dock。
定义平面 / 参考线定义 / 抽线用小窗口，避免挤占钢筋生成器 Dock。
```

小窗口必须支持：

```text
预览
确定
取消
拾取对象
写入工程参考几何对象
```

## 实现边界

UI 层禁止：

```text
直接写钢筋生成算法
直接改 OCCT 拓扑对象当业务真相
把 RebarSmart 弹窗字段删成简化版
让 domain / rebar 直接依赖 AIS / TopoDS / BRep 细节
```

UI 层允许：

```text
展示参数
展示选择状态
调用 command / service / generator
显示预览结果
提交应用 / 取消
显示错误和诊断
```

## 开发入口顺序

建议后续实现顺序：

```text
1. Qt6 主窗口骨架
2. 中央 OCCT Viewer
3. 左侧工程树 / 钢筋树 Dock
4. 右侧 Dock 统一框架和顶部四行摘要
5. STEP 导入与水泥灰显示
6. Viewer 点 / 边 / 面 / 实体选择
7. 钢筋页签第一批外壳按钮
8. FixDistancePanel 或 FixNumberPanel 的第一个完整面板
9. 对应 generator / service / preview 闭环
```

具体钢筋面板开发时，必须先补该面板专项证据文档。

## 自审结果

本轮 UI 文档自审口径：

```text
1. 顶层页签统一为：开始 / 显示 / 钢筋 / 查询 / 工程图。
2. 钢筋页签第一批外壳按钮统一为使用说明“三维布筋”8 个主命令。
3. 扫掠、多段线、螺旋、复制生成、蜗壳等不删除，只是不进入 P0A 验收。
4. 钢筋创建和钢筋组右键编辑统一切换右侧 Dock。
5. 定义平面 / 参考线定义 / 抽线继续使用小窗口。
6. 选择不做独立页签，由 OCCT Viewer + 当前命令 + 状态栏承接。
7. STEP / STP 模型 P0 统一水泥灰显示。
8. 点 / 线 / 面选中为红色，钢筋为橙黄，高亮为洋红。
9. 工程图 / Detail 参考老图石，不把 RebarSmart 钢筋切图作为当前主线。
10. UI 层不写钢筋算法，只调用 generator / service / domain。
```

当前 UI 文档体系只保留有效按钮清单。如需追溯历史讨论，查 Git 记录。
