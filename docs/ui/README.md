# TS_RS UI 文档入口

本文档说明 TS_RS UI 文档的阅读顺序和当前有效口径。

## 当前阶段

当前处于 B 方案：

```text
先收束 UI 可开发规格。
先不进入 C++ 实现。
先把顶层页签、Viewer、右侧 Dock、左侧钢筋树右键、钢筋创建面板规则写清楚。
```

UI 阶段完成后，再进入：

```text
系统架构文档
roadmap / todo.csv
Qt6 + OCCT 工程骨架
RebarSmart 生成器复刻
Detail / 工程图兼容输出
```

## 必读顺序

进入 UI 开发或 UI 方案评审时，先读：

```text
11_UI可开发规格.md
00_UI决策记录.md
01_顶层选项卡与功能矩阵.md
```

再按模块补读：

```text
05_OCCT视图交互与选择设计.md
06_显示颜色与高亮规则.md
07_显示页签图石对齐矩阵.md
09_钢筋页签命令与RebarSmart算法映射.md
10_钢筋创建Dock面板RebarSmart一致性规则.md
03_钢筋树右键菜单设计.md
04_参考面参考线UI设计.md
08_定距钢筋参数面板与IDA证据.md
```


小窗口：

```text

抽线
```

## 开发依据

```text
1. 以 11_UI可开发规格.md 为 UI 开发入口。
2. 钢筋创建页 P0 以 01_顶层选项卡与功能矩阵.md 和 09_钢筋页签命令与RebarSmart算法映射.md 为准。
3. 当前文档体系只保留有效按钮清单。
4. 如需追溯历史讨论，查 Git 记录。
```

## 开发硬边界

```text
UI 层只负责参数展示、选择状态、预览入口、应用 / 取消。
UI handler 不写钢筋生成算法。
钢筋创建参数必须对齐 RebarSmart 对应 GenerateRebarXXXDlg。
工程图 / Detail 输出参考老图石，不使用 RebarSmart 钢筋切图作为当前主线。
```
