# 定数钢筋参数面板与证据

本文档记录 TS_RS `定数钢筋` 右侧 Dock 面板的证据、字段映射和待确认缺口。

核心结论：

```text
定数钢筋不是“定距钢筋改一个输入框”。

RebarSmart 使用说明明确：
  用户选择布筋面，选择两条及以上布筋引导线，
  按用户设置的钢筋根数等分两根引导线，
  再连接等分点生成钢筋。

所以 TS_RS 的 FixNumberPanel 必须把“布筋面 + 多条引导线 + 根数”作为主语义，
不能只复用 FixDistancePanel 的间距优先流程。
```

## PDF 操作语义

使用说明中 `1.1.2 创建定数钢筋` 描述：

```text
在所选择的布筋面上（设置保护层厚度），选择两条及以上布筋引导线，
然后以用户设置的钢筋数等分两根引导线，最后连接两根引导线的等分点，
生成钢筋。
```

操作步骤证据：

```text
1. 在模型上选择布筋面，可多选。
2. 输入保护层厚度。
3. 可点击“反转”按钮反转保护层方向。
```

白话含义：

```text
定数钢筋的核心输入是根数。
引导线至少两条，算法要在引导线上按根数做等分点匹配。
保护层和反向影响布筋面偏移后的实际钢筋位置。
```

## 与定距钢筋的关键差异

FixNumber 相比 FixDistance：

```text
1. INI 为 41 字段，FixDistance 为 57 字段。
2. FixNumber 默认 `distribution.priorityMode = 1`，与“根数优先”语义一致。
3. FixNumber 没有辅助布筋区间字段。
4. FixNumber 没有倍径字段：
   - 切向延长长度倍径
   - 辅助延长长度倍径
   - 弯钩长度倍径
   - 弯钩半径倍径
   这些在 FixDistance.ini 里存在，但 FixNumber.ini 默认不落盘。
5. FixNumber 的 PDF 核心语义是两条及以上引导线等分连接。
```

GAP：

```text
FixNumber.ini 仍保留 `基本参数_间距`。
它可能用于默认值、预览显示、根数反算后的结果、或兼容公共参数结构。
真实用途需要查 FixNumberImp / AfrRebarCore。
```

## 算法语义草案

基于 PDF 和现有逆向分析文档，FixNumber 的高层流程暂定：

```text
输入：
  布筋面
  至少两条布筋引导线
  保护层 / 偏移方向
  左右端距
  钢筋根数
  延长 / 弯钩 / 孔洞参数

流程：
  1. 对布筋面按保护层偏移。
  2. 对每条引导线裁掉左右端距。
  3. 按钢筋根数等分引导线，得到等分点。
  4. 将相邻或配对引导线的等分点连接，形成钢筋轴线。
  5. 应用孔洞、切向延长、辅助延长、端部弯钩。
  6. 可选扫掠成实体钢筋。
```

必须保守：

```text
上面是开发草案，不是最终算法。
真实配对规则、多条引导线排序、闭合线处理、端距调整和孔洞行为，
必须继续用 IDA / 实际运行确认。
```

## 单位规则

沿用 `GenerateRebarData` 字段地图：

```text
DTO 内部长度统一用 m。
DTO 内部角度统一用 rad。
INI Ratio 只用于 UI 显示 / 配置读写换算。
```

已确认：

```text
直径 ratio=1000.0。
保护层、端距、延长、弯钩长度 / 半径 ratio=100.0。
弯钩角度 ratio=57.29577951471995。
```

待确认：

```text
Ratio=100.0 在 RebarSmart UI 上具体显示为 cm，还是工程习惯单位。
```

## 与公共 DTO 的关系

FixNumber 没有明显专属 INI 字段。
首版 DTO 可以使用：

```text
GenerateRebarData
  basic
  solid
  guideSurface
  distribution
  passHole
  tangentialExtend
  auxiliaryExtend
  hook
```

另需生成器上下文保存选择对象：

```text
FixNumberSelectionContext
  selectedFaceRefs
  guideCurveRefs
```

说明：

```text
选择对象不能放进纯参数 DTO。
它们应使用 TS_RS 的 TopologyBinding / ReferenceGeometryBinding。
```
