# 纯算法层实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把 RebarSmart 定距 / 定数钢筋中可脱离 3DE、OCCT、UI 的算法拆成可单测模块，作为后续 FixDistanceGenerator / FixNumberGenerator 的稳定底座。

**Architecture:** 纯算法层只处理数字、字符串、枚举、点位长度数组和诊断，不接触 `TopoDS`、`AIS`、Qt Widget 或 RebarSmart DLL。几何曲线长度、按弧长取点、投影和组装线段放到后续 `IGeometryEngine`，本层只输出“应该在哪里布点、端距如何更新、参数是否合法”。

**Tech Stack:** C++17 / CMake / CTest / 纯标准库；后续代码目录建议在 `app/src/rebarsmart` 和 `app/tests/rebarsmart` 下创建。

```text
AfrRebarCore3D.dll
  0x180044460 GenerateRebarCore::UpdateRebarCountSpaceMargin
  0x180044C90 GenerateRebarCore::UpdateGuideCurveZone
  0x180049EC0 GenerateRebarCore::GetGuideSurfaceOffestDistance

AfrRebarObjImp.dll
  0x180037C20 GenerateRebarFixDistanceImp::FormGuideCurveZoneArray
```

关键事实：

```text
UpdateRebarCountSpaceMargin:
  先 UpdateGuideCurveZone。
  usableLength = curveLength - headZone - tailZone。
  mode == 1 走根数优先。
  mode == 2 走间距列表优先。
  mode == 0 / 默认走间距优先。

UpdateGuideCurveZone:
  head/tail adjust mode 1 -> 0。
  mode 2 -> 0.5 * diameter。
  mode 3 -> 0.5 * diameter + spacing。
  mode 4 -> guide surface offset distance。

GetGuideSurfaceOffestDistance:
  offsetMode 0 / 2 -> cover + 0.5 * diameter。
  offsetMode 1 -> cover - 0.5 * diameter。
  offsetMode 3 / 4 -> firstLayerProtectThicknessNet + 0.5 * diameter。
  offsetMode 5 -> cover。
  result += activeLayerIndex * layerSpacing。

FormGuideCurveZoneArray:
  处理按关键点 / 分段布筋区间时的 head/tail zone 数组。
  涉及引导线反向、孔洞分割、端点 0.5 * diameter 保护性修正。
```

## 公共 DTO 草案

后续实现可按这些名字建类型；本轮只冻结语义，不冻结最终 C++ 头文件细节。

```cpp
enum class DistributionPriorityMode {
    Spacing = 0,
    Count = 1,
    SpacingList = 2,
};

enum class ZoneAdjustMode {
    KeepCurrent = 0,
    Zero = 1,
    HalfDiameter = 2,
    HalfDiameterPlusSpacing = 3,
    GuideSurfaceOffset = 4,
};

struct ZoneLengths {
    double headM;
    double tailM;
};

struct DistributionInput {
    double curveLengthM;
    ZoneLengths zone;
    double spacingM;
    double spacingRatio;
    int count;
    std::vector<double> spacingListM;
    DistributionPriorityMode mode;
};

struct DistributionResult {
    bool ok;
    int count;
    double spacingM;
    double marginM;
    std::vector<double> pointLengthsM;
    std::string diagnosticCode;
};

struct GuideSurfaceOffsetInput {
    int offsetMode;
    int activeLayerIndex;
    double diameterM;
    double coverThicknessM;
    double layerSpacingM;
    double firstLayerProtectThicknessNetM;
};

struct GuideCurveZoneInput {
    ZoneLengths current;
    ZoneAdjustMode headMode;
    ZoneAdjustMode tailMode;
    double diameterM;
    double spacingM;
    double guideSurfaceOffsetM;
};
```

单位规则：

```text
内部长度一律用 m。
内部角度一律用 rad。
INI Ratio 只在配置读写 / UI 显示层使用，不进入纯算法核心。
```

## TODO-007：SpaceListParser

目标：

```text
解析 RebarSmart 的间距列表文本，例如 20*12,25*8。
输出内部米制距离序列，供 PriorityListDistributor 使用。
```

证据：

```text
GenerateRebarFixDistance.ini:
  布筋区间_钢筋间距列表 = 20*12,25*8

docs/rebarsmart/01_GenerateRebarData字段地图.md:
  SpaceListParser 只处理 distribution.spacingList，不处理 UI。
```

输入规则草案：

```text
token 形态：
  N*M 表示数值 N 重复 M 次。
  N 表示单个数值。
分隔符：
  半角逗号 ','。
  全角逗号 '，' 暂按 GAP，首版可返回诊断。
单位：
  列表中的 N 暂按 UI 显示单位进入 parser。
  parser 接收 unitScaleToM 参数，由调用方决定 N 如何转成 m。
```

建议接口：

```cpp
struct SpaceListParseOptions {
    double unitScaleToM;
    bool allowChineseComma;
};

struct SpaceListParseResult {
    bool ok;
    std::vector<double> valuesM;
    std::string diagnosticCode;
    int errorOffset;
};

SpaceListParseResult parseSpaceList(std::string_view text, SpaceListParseOptions options);
```

测试矩阵：

| SL-002 | `20` | 0.01 | 1 个值 0.20 |
| SL-003 | `20*1` | 0.01 | 1 个值 0.20 |
| SL-004 | 空字符串 | 0.01 | `ok=false`, `RS_SPACE_LIST_EMPTY` |

| SL-006 | `20*-1` | 0.01 | `ok=false`, `RS_SPACE_LIST_REPEAT_INVALID` |
| SL-007 | `20*,25` | 0.01 | `ok=false`, `RS_SPACE_LIST_TOKEN_INVALID` |
| SL-008 | `20*12，25*8` | 0.01 | 首版 `ok=false`, `RS_SPACE_LIST_SEPARATOR_UNSUPPORTED` |

## TODO-009：PriorityListDistributor

证据：

```text
UpdateRebarCountSpaceMargin mode == 2：
  读取 RebarSpaceList。
  调用 GetPointLength_ProiriyList。
  count = pointCount。
  margin = curveLength - tailZone - lastPointLength。
```

首版算法策略：

```text
从 headZone 开始累加 spacingList。
第一个点位 = headZone。
后续点位 = 前一点位 + spacingList[i]。
保留不超过 curveLength - tailZone 的点位。
count = 点位数量。
margin = curveLength - tailZone - lastPointLength。
如果点位数量 <= 1，返回诊断。
```

建议接口：

```cpp
DistributionResult distributeBySpacingList(double curveLengthM, ZoneLengths zone, std::vector<double> spacingListM);
```

测试矩阵：
