# TS_RS 系统架构总览

本文档是 TS_RS 的系统架构入口。

## 一句话架构

```text
TS_RS 是 STEP-only 独立工程软件：
Qt6 做界面，
OCCT 做几何 / 显示 / 选择，
RebarSmart 证据驱动钢筋生成逻辑，
老图石 Detail 包负责工程图兼容导出。
```

## 总体数据流

```text
STEP/STP 文件
  ↓
StepImporter
  ↓
ShapeStore + TopologyBinding
  ↓
OCCT Viewer 选择 Face / Edge / Vertex
  ↓
RebarSmart 参数 + 用户输入
  ↓
RebarSmart-style Generator
  ↓
RebarModel
  ↓
AIS 钢筋显示 / ScheduleModel / DrawingModel
  ↓
DetailPackageExporter / Excel-CSV / Rebar STEP
```

## 模块职责

### UI 层

职责：

```text
主窗口
顶层页签
左侧工程树 / 钢筋树
右侧钢筋生成器 Dock
底部消息 / 选择 / 下料表 / 诊断
```

禁止：

```text
写钢筋生成算法。
直接操作 TopoDS 当业务对象。
绕过 command / service 直接改 RebarModel。
删减 RebarSmart 参数字段。
```

### OCCT Viewer 层

职责：

```text
STEP 水泥灰显示
钢筋橙黄显示
点 / 线 / 面红色选中
洋红高亮
旋转、平移、缩放、窗口缩放
X / Y / Z 剖视和剖切面旋转
```

边界：

```text
Viewer 剖视只是显示剖切，不等于工程图剖切。
选择结果必须转换为稳定 TopologyBinding 引用。
```

### STEP / TopologyBinding 层

职责：

```text
导入 STEP/STP。
保存 ShapeStore。
为 Face / Edge / Vertex 生成稳定引用。
支持保存 / 重开后恢复选面、选边、选点。
```

P0 验收：

```text
同一 STEP 多次导入后，关键 Face / Edge 指纹稳定。
钢筋参数引用的是稳定绑定，不是 AIS 临时对象。
```

### Geometry 层

职责：

```text
曲线长度
按弧长取点
切线
投影
偏移
相交
组装 wire
扫掠圆截面
剖切 / HLR 的底层能力
```

边界：

```text
OCCT 细节只能在 geometry/occt 内部。
RebarSmart 生成器依赖 IGeometryEngine，不直接 include TopoDS / AIS。
```

### RebarSmart 核心层

职责：

```text
GenerateRebarData 参数模型
RebarSmart INI 加载
SpaceListParser
PrioritySpaceDistributor
PriorityListDistributor
GuideCurveZoneCalculator
Hook / Extend / PassHole 参数语义
FixDistance / FixNumber / Corner / Tie / Dowel / Frustum / Stirrup / Customize 等生成器
```

证据优先级：

```text
1. 使用说明 PDF / 截图
2. GenerateRebarXXXDlg.CATNls / CATRsc
3. ResourceRebarSmart/*.ini
4. RebarCreation.dll 命令入口
5. AfrRebarObjImp.dll / AfrRebarCore3D.dll / AfrRebarCore.dll 算法函数
6. RebarSmart 实际运行确认
```

当前字段入口：

```text
docs/rebarsmart/00_证据索引.md
docs/rebarsmart/01_GenerateRebarData字段地图.md
docs/architecture/01_纯算法层实现计划.md
```

### RebarModel 层

职责：

```text
保存新系统钢筋业务对象。
承载钢筋组、单根钢筋、线段、弯钩、实体状态、统计字段。
```

禁止：

```text
字段名直接用 StbGroup / StbGeo 作为内部业务真相。
直接依赖 Detail XML。
直接依赖 TopoDS / AIS。
```

### Drawing / Detail 层

职责：

```text
DrawingModel 表达图纸视图、线、剖切、投影、钢筋二维表达、表格。
DetailPackageExporter 将 DrawingModel / RebarModel / ScheduleModel 输出为 Detail 包。
```

边界：

```text
Detail 包是导出格式，不是内部工程格式。
DetailNN.stl 是 XML 文本，不是标准 STL 网格。
```

## 复杂性转移账本

```text
旧复杂性位置：
  3DE / CAA、VisualTS、ACIS、HOOPS、MFC、AutoCAD 插件。

新复杂性位置：
  IGeometryEngine、RebarSmart-style Generator、TopologyBinding、DetailPackageExporter。

收益：
  无商业库运行时依赖，可独立开发和部署。

新增成本：
  需要用 IDA / PDF / 运行确认持续闭合算法和 Detail 字段。

主要失效模式：
  生成结果和 RebarSmart 不一致；
  TopologyBinding 失稳；
  Detail 包 CAD 插件无法导入；
  OCCT 几何容差导致边界差异。
```

## 当前冻结边界

```text
UI 顶层页签和右侧 Dock 规则已冻结。
P0 钢筋创建入口为 8 个三维布筋命令。
运行时商业库依赖为 0。
内部主工程格式是 .tsrebar，不是 Detail 包。
RebarSmart 生成器不能直接依赖 UI 或 AIS。
DetailExporter 不能反向污染 RebarModel。
```
