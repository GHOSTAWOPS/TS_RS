# RebarSmart 单位枚举默认值规则

本文档是 TS_RS 后续 UI 面板、生成器参数 DTO、`.tsrebar` 保存和测试用例的单位 / 枚举 / 默认值消费合同。

## 结论

```text
TS_RS 内部长度统一使用 m。
TS_RS 内部角度统一使用 rad。
UI 显示和 RebarSmart INI 读写必须通过 Ratio 转换。
枚举值在证据未闭合前只允许显示为“模式 N”，不能猜中文名。
默认值优先来自 GenerateRebar*.ini 的 RebarPatternCurObject / TableRow0。
```

白话解释：

```text
代码里不要一会儿米、一会儿毫米、一会儿厘米。
RebarSmart 的 INI 里看起来是 0.032、0.2、0.05，
这些是内部值；界面怎么显示，要看 Ratio。
```

## 证据等级

本文档使用四类等级：

```text
确定：
  来自 INI 字段名、默认值、Ratio、RebarGrade/RebarDiameter/RebarWeight。

高置信：
  来自已复核 IDA 文档或多份文档一致描述，但本轮没有重新打开函数逐行确认。

暂定规则：
  为了保证开发一致性先设为工程规则，但必须保留 GAP。
```

## 全局单位合同

### 内部单位

| 类型 | TS_RS 内部单位 | 说明 | 证据等级 |
|---|---|---|---|
| 长度 | m | 直径、间距、保护层、端距、延长、半径统一按米存储 | 确定 |
| 角度 | rad | 弯钩角度、角筋角度统一按弧度存储 | 确定 |
| 个数 | count | 根数、束内根数、拟合点数、保留段号不做单位转换 | 确定 |
| 比例 | raw ratio | `spacingRatio` 等比例字段原值保存 | 确定 |
| 颜色 | int | RebarSmart INI 默认常见 `255`，真实颜色语义待确认 | 确定字段 / GAP 语义 |
| 线宽 | int | 默认常见 `2` | 确定 |
| 文本 | string | 式样名、等级、间距列表、倍径表达式原样保存 | 确定 |
| 布尔 | bool | `true/false` 原样解析 | 确定 |

### Ratio 消费规则

RebarSmart `GenerateRebar*.ini` 中的数值字段常见 Ratio：

| Ratio | TS_RS 解释 | UI 显示计算 | INI 写回计算 | 证据等级 |
|---:|---|---|---|---|
| `1000.0` | m -> mm | `ui = internalM * 1000` | `internalM = ui / 1000` | 确定 |
| `100.0` | m -> RebarSmart 显示单位 | `ui = internalM * 100` | `internalM = ui / 100` | 确定换算 / GAP 显示名 |
| `57.29577951471995` | rad -> degree | `ui = internalRad * ratio` | `internalRad = ui / ratio` | 确定 |
| `1.0` | 原值 | `ui = raw` | `raw = ui` | 确定 |
| `-1.0` | 不做数值换算 | 字符串 / bool / 文本列表按类型处理 | 原样写回 | 确定 |

重要约束：

```text
Ratio=100.0 的字段不能在代码里直接命名为 mm。
当前只能说“内部米，显示值为 internalM * 100”。

它在 RebarSmart UI 上到底叫 cm、dm，还是无单位工程显示值，
需要实际 3DE 截图或 CATNls/CATRsc 继续确认。
```

```text
  offset = protectThickness + 0.5 * diameter

offsetMode == 1:
  offset = protectThickness - 0.5 * diameter

offsetMode == 2:
```

## 材料 catalog 规则

### 等级

`RebarGrade.ini` 提供等级代码：

```text
A, B, C, D, E, G, H, I, HA, DX,
#3, #4, #5, #6, #7, #8, #9, #10,
O, P, Q, R, S, T
```

TS_RS 规则：

```text
等级按字符串代码保存。
UI 用下拉选择。
生成器不解释等级力学含义。
下料 / Detail / Excel 需要时再映射显示。
```

### 直径

`RebarDiameter.ini` 分组：

```text
BarCHN:
  5, 6, 6.5, 8, 8.2, 10, 12, 14, 16, 18, 20, 22, 25, 28, 32, 36, 40

BarASM:
  10, 13, 16, 19, 22, 25, 29, 32

BarEUR:
  8, 10, 12, 14, 16, 20, 25, 28, 32

Strand:
  5, 7, 9, 9.5, 12.7, 15.24, 17.8, 21.6

Anchor:
  40
```

TS_RS 规则：

```text
Catalog 显示单位按 mm。
DTO 保存为 m。
例如 UI 32 -> internal 0.032。
```

注意：

```text
原 INI 中 Strand 存在 `9,5` 这种逗号小数字符串。
TS_RS catalog parser 必须兼容区域小数写法，不能简单用英文逗号 split 全部场景。
```

### 重量

`RebarWeight.ini` 按直径给出重量表，单位未在 INI 中声明。

TS_RS 规则：

```text
先作为 raw catalog weight 保存。
下料表计算前必须确认单位，不能直接假设 kg/m 写入正式输出。
```

高置信推断：

```text
BarCHN 32 -> 6.313 与常见钢筋理论重量 kg/m 接近。
但本文档不把它写成最终单位事实。
```

## 生成器默认值入口

```text

3. Default/GenerateRebar*.ini 的 TableRow0。
4. IDA 初始化函数默认值。
5. TS_RS 代码兜底默认值。
```

## UI 面板消费规则

UI 面板必须消费本文档，不直接解释 INI：

```text
GenerateRebar*.ini
  -> RebarSmartResourceReader / ParamCatalog
  -> 本文档定义的单位合同
  -> GenerateRebarData / 专属 Params
  -> Dock 控件显示
```

禁止：

```text
UI 控件直接把 0.032 当 32 显示。
UI 控件直接把 Ratio=100.0 写死成 mm。
UI 根据字段名猜枚举中文名。
UI 为了好看删除 RebarSmart 字段槽位。
UI 把 TopoDS / AIS 对象保存进业务参数。
```

允许：

```text
未确认枚举显示“模式 0 / 模式 1”。
未确认算法字段显示为禁用或“待复刻”，但 DTO / 保存结构保留。
开发期在 Dock 中显示 internal value 和 Ratio，方便排查。
```

## 测试用例要求

后续涉及参数读取 / UI 显示 / 保存写回时，至少覆盖：

```text
1. diameter 0.032 + Ratio 1000.0 -> UI 32。
2. hook angle 1.57079632679 + Ratio 57.29577951471995 -> UI 90。
3. Ratio -1.0 字段不做 double 解析，bool/string/list 原样按类型处理。
4. Ratio 1.0 枚举字段不做长度单位转换。
5. Ratio 100.0 字段 round-trip 后 internal 值不变。
6. Strand `9,5` 不能被普通 CSV split 破坏。
7. unknown enum value 只能显示模式 N，不能崩溃或猜名。
```
